## 🔗 Workflow: PR → Issue Linking

### Bước 1: Parse Issue Number từ Branch Name

```bash
BRANCH=$(git branch --show-current)
# Ví dụ: feature/1_push-code → Issue #1
```

**Supported Patterns:**

| Pattern | Example | Issue Number |
|---------|---------|--------------|
| `*/{number}_*` | `feature/1_push-code` | `1` |
| `*/{number}-*` | `feature/125-add-search` | `125` |
| `*/*-{prefix}{number}*` | `fix/ABC-456-fix-bug` | `456` |
| `*/issue-{number}*` | `fix/issue-789-update` | `789` |

### Bước 2: Fetch Issue từ GitHub

```bash
curl -sS \
  -H "Authorization: Bearer ${GITHUB_TOKEN}" \
  -H "Accept: application/vnd.github+json" \
  "https://api.github.com/repos/PersonalProjectJob/Job360/issues/${ISSUE_NUMBER}"
```

**Lấy thông tin:**
- Issue title
- Issue body (first 3-5 lines)
- Issue URL

### Bước 3: Include Issue Link trong PR Body

```markdown
## Linked Issue (Development)
- **Issue:** #1
- **Title:** Push code to repository
- **Link:** https://github.com/PersonalProjectJob/Job360/issues/1
- **Status:** This PR implements and resolves the linked issue
```

### Bước 4: Use GitHub Closing Keyword

```markdown
## Description
This PR implements the feature described in issue #1.

**Closes #1**
```

**Supported Keywords:**
- `Closes #<number>` - Đóng issue khi PR merge
- `Fixes #<number>` - Tương tự Closes
- `Resolves #<number>` - Tương tự Closes
- `Refs #<number>` - Chỉ reference, không đóng

## 🚀 Complete PR Creation Example

### Scenario: Feature Branch `feature/1_push-code`

```bash
#!/bin/bash

# Step 1: Get branch name
BRANCH=$(git branch --show-current)
echo "📌 Branch: $BRANCH"

# Step 2: Parse issue number
ISSUE_NUMBER=$(echo "$BRANCH" | grep -oP '^\w+/(\d+)' | grep -oP '\d+$')
if [ -z "$ISSUE_NUMBER" ]; then
  echo "❌ Could not parse issue number from: $BRANCH"
  exit 1
fi
echo "📌 Found issue: #$ISSUE_NUMBER"

# Step 3: Fetch issue from GitHub
ISSUE=$(curl -sS \
  -H "Authorization: Bearer ${GITHUB_TOKEN}" \
  -H "Accept: application/vnd.github+json" \
  "https://api.github.com/repos/PersonalProjectJob/Job360/issues/${ISSUE_NUMBER}")

ISSUE_TITLE=$(echo "$ISSUE" | jq -r '.title')
ISSUE_BODY=$(echo "$ISSUE" | jq -r '.body' | head -3)

echo "📌 Issue: #$ISSUE_NUMBER - $ISSUE_TITLE"

# Step 4: Determine base branch
if [[ "$BRANCH" == hotfix/* ]]; then
  BASE_BRANCH="main"
else
  BASE_BRANCH="test"
fi
echo "📌 Target: $BASE_BRANCH"

# Step 5: Push branch
git push -u origin $BRANCH

# Step 6: Create PR with issue linking
gh pr create \
  --head $BRANCH \
  --base $BASE_BRANCH \
  --title "$BRANCH: Implement $ISSUE_TITLE" \
  --body "## Summary

**Branch:** \`$BRANCH\`
**Issue:** #$ISSUE_NUMBER - $ISSUE_TITLE

This PR implements the feature described in issue #$ISSUE_NUMBER.

## Linked Issue (Development)
- **Issue:** #$ISSUE_NUMBER
- **Title:** $ISSUE_TITLE
- **Link:** https://github.com/PersonalProjectJob/Job360/issues/$ISSUE_NUMBER
- **Status:** This PR implements and resolves the linked issue

## Issue Description
$ISSUE_BODY

## Changes Made
- Implement feature per issue #$ISSUE_NUMBER

## Testing
- [ ] Pre-commit tests passed
- [ ] Manual testing completed

Closes #$ISSUE_NUMBER

## Refs
- Issue: #$ISSUE_NUMBER
- Branch: \`$BRANCH\`"

# Step 7: Auto-assign
PR_NUMBER=$(gh pr view --json number -q .number)
gh pr edit $PR_NUMBER --add-assignee PersonalProjectJob

echo "✅ PR created: #$PR_NUMBER"
echo "📌 Linked to issue: #$ISSUE_NUMBER"
echo "🔗 PR will auto-close issue #$ISSUE_NUMBER when merged"
```

## 📊 GitHub Issue "Development" Section

Khi PR được tạo với keyword `Closes #<number>`:

GitHub sẽ tự động hiển thị:

```
## Development
- Successfully merging this pull request may close these issues.
  - #1 - Push code to repository
```

**Link trong Issue:**
- Issue page sẽ show PR trong "Development" section
- Click vào PR để xem chi tiết
- Status shows khi PR chưa merge

## ⚠️ Important Notes

### 1. Branch Naming is Critical
Branch name PHẢI có issue number:

✅ **Đúng:**
- `feature/1_push-code`
- `bugfix/2_fix-login`
- `hotfix/issue-789-security`

❌ **Sai:**
- `feature/push-code` (không có number)
- `fix-login-bug` (không có number)
- `test-branch` (không có number)

### 2. GitHub Keywords là Case-Insensitive
```markdown
Closes #1    ✅
closes #1    ✅
CLOSES #1    ✅
```

### 3. Multiple Issues
Nếu PR liên quan đến nhiều issues:

```markdown
Closes #1
Refs #2, #3
```

### 4. Cross-Repository Issues
```markdown
Closes PersonalProjectJob/Job360#1
```

## 🔧 Troubleshooting

### Issue: "Could not parse issue number"

**Nguyên nhân:** Branch name không có pattern đúng

**Giải pháp:** Rename branch với pattern `type/number_description`

```bash
git branch -m old-name feature/1_new-name
```

### Issue: "GitHub API returns 404"

**Nguyên nhân:** Issue không tồn tại hoặc token không có quyền

**Giải pháp:**
1. Check issue exists: `https://github.com/PersonalProjectJob/Job360/issues/<number>`
2. Verify token has `issues:read` scope
3. Try without token (public repo)

### Issue: "PR doesn't close issue"

**Nguyên nhân:** Keyword không đúng hoặc ở sai vị trí

**Giải pháp:**
1. Dùng `Closes #<number>` (not `Close` hay `Closed`)
2. Đặt trong PR body (không phải title)
3. Check spelling và spacing
